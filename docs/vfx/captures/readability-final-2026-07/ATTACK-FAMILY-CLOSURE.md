# Attack-family technical closure record

**Status:** direct headed-compositor proof now covers every current damaging
player source and authored Mythic treatment. This is a technical source and
rendering closure, not a claim that people can identify every effect without
training.

## Evidence rule

Every row below has two independent parts:

- A real production-preview run selects its trait or fusion through visible DOM
  controls, then waits for the exact public source event or immutable snapshot
  before taking compositor stills.
- The exhaustive source-to-treatment contract is covered by
  `illustrated-vfx-presentation.test.ts`, `trait-command-presentation.test.ts`,
  `attack-vfx-palette.test.ts`, and `vfx-easing.test.ts`. Those tests guard
  source identity, palette ownership, hero cast cues, bounded concurrency, and
  fade discipline; they do not replace the visual stills.

The older interrupted full-family run and aggregate reports with unrelated
blocked targets are retained as history. A parent report with `pass: false` is
never presented as a passing matrix below; where used, only its explicitly
recorded target-level `PASS` row is cited and a newer focused report is
preferred whenever available.

## Capture integrity notes

- The successful captures are headed Chromium production previews and use only
  normal autoplay plus visible DOM card/fusion choices. Their reports record
  source IDs, tick windows, phases, and source manifests.
- Some reports contain a generic missing-resource `404` browser message. It is
  retained in the JSON evidence; this record does not claim a clean console.
- A few failed seed attempts are intentionally retained beside their successful
  alternate-seed replacements. They are evidence that the harness does not
  manufacture a trait, fusion, or renderer event when a real run cannot reach
  it.

## S7 technical conclusion

The renderer's current damaging source set has both exhaustive route tests and
at least one direct real-run compositor lifecycle per source/treatment above.
Together with the normal-run matrix, this closes **S7 as technical visual
evidence**. Whether players can recognize, prioritize, and enjoy these effects
remains a separate three-person human-review decision in
[the owner checklist](../../../playtests/visual-readability-owner-checklist.md).

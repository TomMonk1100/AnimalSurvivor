# Gate 0: Concept, License, and Art Test

Gate 0 tests whether “the animal's body is the loadout” is attractive and
understandable before gameplay production begins.

## Deliverables

- [x] Three visual-direction boards with before/mid/after hero transformations
- [x] One 15-second animated concept mockup
- [x] Twelve attachment cards and six paired mythic evolutions
- [x] Audited shortlist of CC0 hero, enemy, environment, UI, and audio sources
- [x] Initial [`assets/ASSET_LEDGER.md`](../../assets/ASSET_LEDGER.md)
- [x] Landing-page A/B pitch test
- [x] Five competitor teardown notes
- [x] Ten-person qualitative test script and results template
- [ ] Gate 0 go/revise/stop report

## Pass rule

Gate 0 passes only when:

1. target players can repeat the product hook without prompting; and
2. at least 6 of 10 testers prefer a visible adaptation over a mechanically
   equivalent invisible stat increase.

Positive reactions to art alone do not pass the gate. Testers must understand
that attachments are the build system and expect the final animal to remain
recognizable.

## Owner input in this gate

Gate 0 uses two of the four owner contributions defined by the main plan:

1. choose the first three heroes and give each one a name plus a one-sentence
   personality;
2. select one of three generated visual-direction boards.

The other two owner contributions happen later: selecting a mascot feature from
AI-generated options and choosing or renaming five final evolutions.
